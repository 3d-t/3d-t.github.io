Zwave

This integration uses a script for subscribing and publishing to an MQTT server connected to the Zwave2Mqtt bridge. The integration is bi-directional, which means you can read values but also set values, for example to turn on a switch.

Usage: Read and control wireless Zwave devices such as sensors and controllers.

Secret#

Create a new secret but instead of setting the type to mqtt we now set the type to script and let the script handle the MQTT interaction.

Script#

The Zwave2MQTT bridge returns the value in a JSON object with the format {"value": 100, "time": 1653474051550} The script gets the last value from the MQTT topic and returns an object with the topic, value and time of the last message.

class Zwave {      init(context){this.context = context}
    async get(object) {        var event = await mqtt.get(object)        return {object: object, value: event.message.value, time: event.time}    }        async set(object, key, value){        var topic = object + '/set';        var message = '{"value":' + value + '}';        mqtt.publish(topic, message, this.context)    }}

To test the script enter the code below and select Run.

let zwave = new Zwave()zwave.init(await encryption.secret("ZwaveSwitch"))let event = await zwave.get('zwave/office/lamp/on')print('State ' + JSON.stringify(event))await zwave.set('zwave/office/lamp/on','', false)

Importer#

The importer shows how three topics are mapped to MQTT topics. Note that the formula field has an expression that converts the true or false values to 0 and 1.

Exporter#

To send messages back to the MQTT server select Exporters from the menu and create a new exporter. In the mapping the value of the field on is converted back to true or false and mapped to the MQTT topic on.

To test the connection select States from the menu and search for the state of topic office.htc.lamp.on and change the value from 0 to 1 or vice versa. The switch should then be turned on or off.
