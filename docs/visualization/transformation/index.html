Transformation

In this section, we explain how you can apply transformations to 3D objects and how you can assemble a digital twin by combining separate GTLF models. Transformations on 3D objects can be used to visualize the state of an object depending on IoT parameters. For example, you can show the location of an elevator in a building or the position of the arm of a robot in a factory.

Solids​

This example shows how IoT parameters stored in topics can be used to translate and rotate objects in a small crane, or cherry picker. The body of the crane is constructed from a block and a raised cylinder near the edge as shown below. The two objects are shrink-wrapped into a new body using the Hull operation.

The crane object includes a part Arm that can be rotated in two directions and that can be extended so that the grey inner shaft moves out of the yellow outer shaft.

Topics​

The example shows a model with a crane in a parking place. The topic for the crane is crane.T01. You can move the crane by specifying a translation subtopic with the x-, y-, and z-values for the translation relative to the origin. The topic for the second crane points to the same GLTF model and does not have any translation so it will be positioned at the origin.

parking.T01.geometry = Parking.gltf
crane.T01.translation = 10, 0, 0
crane.T01.geometry = Crane.gltf

You can now rotate the part Arm of the crane assembly by creating a subtopic arm.node that links to the part that should be rotated and a subtopic arm.node that specifies the amount of rotation in x-, y-, and z-direction.

crane.T01.arm.node = Arm
crane.T01.arm.rotation = 0, 30, 45

The Arm part includes another part Extension that can move in and out of the yellow arm. For this node, you can set the extension by setting the y-value to 3. 

crane.T01.extension.node = Extension
crane.T01.translation = 0, 3, 0

Model​

Create a new model and include the topics for the parking and the crane. The transformations defined in the subtopics for the first crane are applied so that the crane is moved on the parking, the arm is rotated counterclockwise, and is rotated upwards. The arm extension with the basket is moved out 3 units. If you click on the crane you see the current values in the topics. If the values change, the model will be adjusted. In this example, we have not created a separate form for the crane. If you click on the crane all topics related to the crane are displayed. The values can be changed by setting up a real-time connector to the actual device or by editing the values manually.

Explanation​

A GLTF model has an internal hierarchy, a so-called tree of nodes. When applying a translation or rotation to a node all child nodes are translated or rotated as well. For example, when a translation is applied to the whole model is moved. When a rotation is applied to the arm, the extension and basket are also rotated because they are part of the arm. Both cranes have a link to the same GTLF model and crane_1 also has additional transformation to certain nodes in the model. Note that the topics parking.T01 and crane.T01 each point to a separate GTLF model.
